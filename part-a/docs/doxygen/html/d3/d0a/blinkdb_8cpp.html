Detailed Description

Implementation of the BlinkDB key-value storage engine and REPL.

This file implements the core functionality of the BlinkDB storage engine, including the Bloom filter, in-memory indices for SSTables, and helper functions for memory mapping. It also contains the main REPL for interacting with the storage engine.

Perform a binary search on the in-memory index for a given SSTable file.

First, the Bloom filter is checked to quickly rule out a missing key. If the key might exist, a binary search is performed on the index and the corresponding record is retrieved.

Parameters
filenameThe name of the SSTable file.
keyThe key to search for.
Returns
The value associated with the key, or "NULL" if not found.

Build an in-memory index for an SSTable file and create its Bloom filter.

The function reads each record (line) from the SSTable, extracts the key, and records its file offset. It then builds a Bloom filter based on the keys.

Parameters
filenameThe name of the SSTable file.
Returns
A vector of (key, file offset) pairs sorted by key.
